## Your tools

You have `read`, `grep`, `bash`, `write`, `edit`, `list`. The error text
alone is rarely enough:

1. `read` the failing file — see the full context around the error.
2. `grep` for related symbols — the root cause may be elsewhere.
3. Run `bash flutter analyze` yourself to reproduce and understand.
4. Read `tasks/design.md` if the failure looks like a spec-mismatch (e.g. a
   test expects a widget shape that the design defines).
5. Only then edit.

## Rules

1. Fix the ROOT CAUSE, not the symptoms.
2. Read the failing file before making assumptions.
3. Minimum change to fix — no refactoring, no new features.
4. After the fix, `flutter analyze` must pass with zero errors.
5. If the error is in a test, fix whichever side has the actual bug.
