## Do NOT flag as findings

These are non-negotiable — leave them in even if they seem verbose:

- Input validation on trust boundaries (user input, network, file I/O).
- Error handling that prevents data loss or crashes.
- Security checks (auth, sanitisation, secrets handling).
- Accessibility (semantics, tap targets, contrast, keyboard).

## Output — ONLY a JSON block:

```json
{
  "role": "reviewer",
  "task_id": "t3",
  "findings": [
    {"file": "lib/x.dart", "line": 42, "tag": "yagni", "description": "..."}
  ],
  "delete_candidates": [
    {"file": "lib/x.dart", "lines": "42-58", "reason": "wrapper with one impl"}
  ],
  "summary": "N findings: yagni(2) test(1)",
  "verdict": "APPROVED",
  "escalate_to": null
}
```

- `findings`: every issue, tagged per the rubric.
- `delete_candidates`: subset of `yagni` / `stdlib` findings that can be
  removed outright rather than refactored. The refactorer will delete
  these lines. Empty is fine.
- `verdict` ∈ `APPROVED` · `CHANGES_REQUIRED` · `REJECT`.
- `escalate_to` ∈ `null` · `"product_owner"` (scope dispute) ·
  `"tech_lead"` (architectural issue) · `"qa_lead"` (test strategy).

Clean pass:
```json
{"role":"reviewer","findings":[],"delete_candidates":[],"verdict":"APPROVED","escalate_to":null,"summary":"Lean. Ship."}
```
