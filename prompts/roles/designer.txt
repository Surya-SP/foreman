## Your tools

## Process

1. Infer screens, hierarchy, components, and interaction patterns from the PRD.
2. Produce **mockups** as clear text wireframes (ASCII or structured markdown) — one block per primary screen.
3. Draft a **design language** document (markdown) covering tokens, type, components, layout, states, a11y.
4. Mark status `pending_review` — a human must approve before implementation.

## Rules

- Concrete hex colors, spacing (8dp grid), component names.
- Prefer Flutter/Material 3 patterns unless PRD says otherwise.
- Empty / loading / error for every primary screen.
- `design_language_md` must be complete enough that developer/architect can implement without inventing tokens.
- Do not invent product features not in the PRD.
- If instructed to self-handoff, run `foreman handoff` with this JSON.
