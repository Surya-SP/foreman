## Non-negotiable craft (anti AI-slop)

**Ban unless PRD explicitly demands:**
- Purple/violet gradient heroes, glassmorphism everywhere, neon, skeuomorphism
- Generic “SaaS Inter-on-white” with no product personality
- Equal-weight CTAs (every button primary)
- Decorative blobs, stock “happy people” metaphor
- More than one competing accent on a screen
- Tiny grey-on-grey text; nested card stacks; endless shadows
- Fake metrics / lorem when PRD has real labels

**Do instead:**
- **One primary action per screen** — kit `ButtonVariance.primary`; secondaries quieter (secondary/outline/ghost)
- **Surfaces need rest** — spacing scale only: 4 8 12 16 24 32 48 64
- **Radius scale only:** 12 16 20 24 (+ pill when needed)
- **Semantic color roles** with concrete hex (light + dark). Default accent if unset: `#3B82F6`. Neutral grayscale surfaces. Dark-mode first.
- **Type scale with purpose** — display / headline / title / body / label
- **shadcn_flutter components** — `ShadcnApp`, kit `Scaffold`/`AppBar`, `Button`, `Card`, `TextField`, `NavigationBar`/`Tabs`, `Toast`/`AlertDialog`, etc.
- **Material only** when no kit equivalent and you document why
- **Empty / loading / error** for every primary screen
- **A11y** — 48×48 targets, WCAG AA, no color-only meaning
- **Motion** — ~200ms, kit animation system; reduce-motion

## Process

1. Extract users, JTBD, platforms, constraints from PRD.
2. Define IA: primary screens + navigation pattern (kit nav components).
3. Token system: color roles + hex (light/dark), type, space, radius, motion.
4. Wireframe each primary screen (regions, hierarchy, primary CTA).
5. Map every control to a **documented** shadcn_flutter component (or justified Material exception).
6. States, a11y, do/don’t, anti-slop checklist.
7. `status` = `pending_review` — human approves before implementation.

## Rules

- Concrete hex in `token_index` AND markdown (include dark surfaces).
- No invented product features.
- Component names must exist in shadcn_flutter docs/kit — never hallucinate.
- Prefer kit names implementers can code: `Button` + `ButtonVariance.*`, `Card`, `ShadcnApp`, …
- If self-handoff is instructed, run `foreman handoff` with this JSON as the final step.
